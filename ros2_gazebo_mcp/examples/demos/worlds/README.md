# Demo World Files

This directory contains world files (SDF format) generated by the demonstration scripts.

## 📁 Generated Worlds

World files are created when you run demonstrations with export enabled:

```bash
# From examples/demos/ directory:
python3 01_complete_navigation_demo.py
# Creates: navigation_course.sdf

python3 06_world_generation_showcase.py --export-all
# Creates: Multiple world files showcasing different features
```

## 🌍 World Files

### Navigation Worlds
- **navigation_course.sdf** - Maze obstacle course for robot navigation

### Feature Showcase Worlds
- **maze_world.sdf** - Maze pattern obstacles
- **grid_world.sdf** - Grid pattern obstacles
- **circular_world.sdf** - Circular pattern obstacles
- **volumetric_world.sdf** - Volumetric lighting demonstration
- **shadows_*_world.sdf** - Shadow quality comparisons (low/medium/high/ultra)
- **animation_world.sdf** - Animation system showcase
- **trigger_world.sdf** - Trigger zones demonstration
- **environment_world.sdf** - Environmental effects (fog, wind)

### Benchmark Worlds
- **benchmark_*_world.sdf** - Reproducible benchmark worlds with different seeds

### Interactive Worlds
- **interactive_world.sdf** - World created through interactive demo

## 🚀 Using World Files

### View in Gazebo

```bash
gazebo navigation_course.sdf
```

### Launch with ROS2

```bash
ros2 launch gazebo_ros gazebo.launch.py world:=navigation_course.sdf
```

### Import into Your Project

```bash
cp navigation_course.sdf /path/to/your/project/worlds/
```

## 📝 World File Format

All files are in SDF (Simulation Description Format):
- Standard Gazebo format
- XML-based structure
- Human-readable
- Version-controlled friendly

## 🧹 Cleanup

To remove all generated worlds:

```bash
rm *.sdf
```

To keep specific worlds and remove others:

```bash
# Keep navigation_course.sdf, remove others
ls | grep -v "navigation_course.sdf\|README.md" | xargs rm
```

## 🔄 Regeneration

All worlds can be regenerated by running their respective demos:
- Reproducible with same parameters
- Benchmark worlds use seeds for exact reproduction
- No risk of losing world designs

## 📊 World Statistics

Typical world file sizes:
- Simple worlds: ~2-5 KB
- Obstacle courses: ~10-30 KB
- Feature-rich worlds: ~20-50 KB
- Complex animations: ~30-100 KB

## 💡 Tips

1. **Version Control:** World files are text-based and work well with git
2. **Customization:** Edit SDF files directly for fine-tuning
3. **Sharing:** Export worlds to share exact simulation environments
4. **Testing:** Use benchmark worlds for consistent testing scenarios
